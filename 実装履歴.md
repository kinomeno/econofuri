## 重要な技術判断

| 判断 | 理由 |
|---|---|
| **pdf.js でPDFをCanvas化** | pdf-lib で生PDFを再利用するとフォント・回転・サイズの破綻が多い。Canvas化で全部「画像」として一律処理 |
| **裏ダミーは Canvas プログラム描画** | 画像素材を持たないため軽量、無限バリエーション、著作権安全、実データ混入リスク0 |
| **裏ダミーエンジンを独立モジュール化** | Lv2.5（ブラウザ拡張）／Lv3（exe）で共有するため。`window.EconofuriDummyEngine` をエクスポート |
| **A4 96dpi（794×1123）でCanvas化** | 解像度と速度のバランス。150dpi だと重い |
| **印刷は `window.print()` まで** | ブラウザの制約（両面・給紙トレイ指定は不可）。仕様書のLv2要件「ダイアログを開くところまで」に一致 |
| **alpha 全濃度 0.95 統一**（Ver0.07〜） | 黒インクの濃さで段階分けすると「使い古された薄い紙」になり、コンセプト「ボツ書類の裏面再利用」と矛盾 |
| **段階＝要素量＋失敗演出数**（Ver0.07〜） | alpha統一の代わり。light=失敗0／normal=失敗1／heavy=失敗2＋重ね1／extreme=失敗5＋重ね2 |
| **`?v=Ver番号` キャッシュバスト**（Ver0.03〜） | python http.server がキャッシュさせる。実機で「古い辞書を握ったまま」のバグに遭遇した |
| **プレビューは表＋裏Canvas横並び**（Ver0.10〜） | iframe では「表裏が同時に見えない」のでユーザー不満。Canvas直描画でラベル付き並べた |
| **landing縦スクロール無し化**（Ver0.10〜） | デスクトップ前提。`body[data-app-state="landing"] { overflow: hidden }` ＋要素圧縮。モバイルはスクロール許容 |
| **裏ダミー地色を純白に**（Ver0.12〜） | クリーム色 `#FAFAF7` だと「印刷したらインクの無駄」とユーザー判断 |

## 詰まったポイントと回避策

| 問題 | 原因 | 回避策 |
|---|---|---|
| **ブラウザが古いJS/CSSを使う** | python http.server がキャッシュ許可 | `<link>` `<script>` に `?v=0.NN` クエリを付け、Ver更新時に揃える |
| **preview_screenshot がタイムアウト** | プレビューレンダラの不安定（Canvas多用で重い） | `preview_eval` で DOM／値を直接チェックして検証する |
| **preview_eval 内の `click()` が効かない時がある** | `<details>` が閉じてると child の click が空振り、または非同期で完了前に確認 | `details.open = true` を先に、または十分な setTimeout 待機（3〜4秒） |
| **smooth scroll が preview で動かない** | ヘッドレスChromeで `behavior:'smooth'` が無効 | 実機では動くので問題なし。preview検証では `scrollY` チェックを諦める |
| **Canvas を toBlob → pdf-lib に渡すと重い** | A4を150dpiで作ると 1240×1754 で約500KB×ページ数 | 96dpi（794×1123）に縮小、PNG圧縮、それでも狂気モードは1MB超になる |

## 設計上の境界

- **無料／有料のコード分離**：境界は localStorage のパスワード判定のみ。`dummy-engine.js` の `PATTERNS` 配列で「無料パターン」と「有料パターン」を分離する想定（現状は未分離、全部無料に出ている）。設定パネル（#7）実装時に整える
- **ファイル外部送信なし**：全処理ブラウザ内。`fetch` は一切しない（pdf.js worker のロードのみ）
- **シード固定で同じ結果**：`mulberry32(seed)` を使用。生成結果の再現性確保
