## 公開時の流れ（記録）

### A. GitHub にリポジトリ作成 → push

1. GitHub で `econofuri`（仮）リポジトリを作成（public）
2. `git -C エコノフリ remote add origin https://github.com/kinomeno/econofuri.git`
3. `git -C エコノフリ push -u origin main`

### B. Vercel デプロイ

1. Vercel に GitHub アカウントでログイン
2. New Project → econofuri リポジトリを Import
3. Framework Preset = `Other`、その他はデフォルト（`vercel.json` で `outputDirectory: app` を読む）
4. Deploy
5. デフォルトURL：`https://econofuri.vercel.app/`（仮、変更可能）

### C. GitHub Pages（代替）

1. リポジトリ Settings → Pages
2. Source = `Deploy from a branch`、Branch = `main`、Folder = `/app`
3. URL：`https://kinomeno.github.io/econofuri/`

## 公開URL確定後に差し替える箇所

公開URLが確定したら、以下の `（URL差し替え）` を実URLに：

| ファイル | 行 | 内容 |
|---|---|---|
| `app/index.html` | OGP `og:image` `twitter:image` | 相対 `og-image.png` → 絶対URL推奨 |
| `app/index.html` | OGP `og:url` 追加 | `<meta property="og:url" content="https://...">` |
| `README.md` | 「公開URL」セクション | 実URLへ |
| `note/初回公開記事.md` | 「公開URL」「本体」 | 実URLへ |
| `note/初回公開記事.md` | 「ソースコード」 | GitHubのURL |

`app/js/main.js` の `updateShareXUrl()` は `window.location.origin + pathname` で動的に取得するので、デプロイ先で自動的に正しいURLになります（修正不要）。
