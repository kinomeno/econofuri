- 製品名：**エコノフリ**（英字：**eco nofuri**）
- 一言：印刷したいファイルを投げ込むと、表＝本来の書類／裏＝裏紙っぽいダミーが刷られた、表裏交互PDFを書き出すブラウザアプリ

- 作者：キノメノ／Kinomeno

## 3. 採用済みの開発方針（要遵守）

ユーザーとの相談で確定。chatGPT意見の推奨を全採用。

### 3.1 実装スタンス
- **「確実に動く最小構成」を最優先**。PDF完全再現や高度な印刷制御より、「ファイル投入 → 表裏PDF生成 → 印刷できる」体験を優先。
- UIは後から増築しやすい構造で組む。
- 暴走実装（凝りすぎ・先回り抽象化）を避ける。

### 3.2 PDF処理
- **PDFは pdf.js でレンダリングし、Canvas画像として pdf-lib へ再配置してよい**。完全なPDF構造維持は不要。
- これによりサイズ違い・回転・埋め込みフォント・異常PDFの問題を回避。

### 3.3 裏紙ダミー
- Canvas描画でプログラム生成（画像素材を持たない）。
- **擬似日本語**で「読めそうで読めない」内容にする。実在の固有名詞・実データは生成しない。
- **ダミー濃度スライダー**を実装：
  - 薄い：ほぼ白紙、うっすら罫線
  - 普通：一般的な裏紙
  - 濃い：会議資料っぽい
  - 狂気：グラフ・スタンプ・表まみれ（SNS映え狙い）

### 3.4 印刷
- 現実ライン上限：**PDF生成 → blob URL → 新規タブで開く → `window.print()`**。
- Lv2（印刷ダイアログ自動起動）は「開くところまで」を上限とし、両面・給紙トレイ指定は試みない。

### 3.5 カモフラージュUI
- 印刷履歴風ログを擬態UIに流す（実動作のログを流用）。例：
  ```
  [完了] 見積_042.pdf  A4 / 両面 / eco mode
  [完了] 会議資料.pdf  A4 / recycle safe
  ```
- Esc 2連打で即「スプレッドシート風」に切替（緊急ホットキー）。
- **小窓モード**（ボタン名「小窓表示」）：
  - ブラウザウィンドウそのものの縮小は**現代ブラウザのセキュリティ制約で不可**（`window.resizeTo()` は自分で `window.open()` したウィンドウにしか効かない）。
  - 代わりに **CSSでアプリUI自体を画面右下の小さなパネル（目安200×300px）に縮める** 形で実装。
  - 背景はぼかし／透明／擬態モードのいずれか。
  - 再度ボタンで元サイズに戻す。
  - 仕様書の「小窓貼り付けの簡易版」の意図を維持。
  - 採否は実装後の使用感で最終決定（ユーザー判断）。

### 3.6 言語
- **全UI日英切替**（タイトル含む全訳。「最小英語化」案から方針変更）。
- 言語切替ボタンを画面右上付近に設置（日／EN）。
- 開始時に利用者の言語を自動判定（`navigator.language`）し、日本語以外は英語表示。切替ボタンで明示変更可能。
- 文字列は一箇所（例：`i18n.js` のキー辞書）にまとめる。
- 英語版の固有名詞例：製品名は「eco nofuri」、キャッチは「My safety-managed printing」など（暫定。実装時に最終決定）。

### 3.7 マネタイズ境界
- 無料／有料の境界を**コード上で明確に分離**（後から課金方式を変えられるように）。
- 有料解放：業界別ダミー、有料画面モード（タスク管理風／メールクライアント風／ファイル転送ツール風 等）。
- パスワード解放状態は **localStorage** で保持（サーバー認証なし）。
- ゲスト購入のメール紛失リスクに備え、入力済みパスワードは端末に確実に保存し再入力不要にする。

### 3.8 画面デザイン方針（chatGPT案ベース）

トップ画面はchatGPT作成のデザインイメージを採用。骨子：
- 緑系トーン（「いかにもecoぶってる」自虐とマッチ）
- 初回中央に大ロゴ「エコノフリ／eco nofuri／私の安全管理印刷」 → ファイル投入後は左上に縮小
- 中央大の説明文「印刷したいファイルを入れるだけ。表はそのまま、裏はいかにも裏紙っぽく。」
- 大きなドロップエリア（点線枠、雲アップロードアイコン）
- 「エコノフリでできること」3カラム説明（表／裏／表裏交互PDF）
- 下部固定のプライバシー明示バー（「すべてブラウザ内で処理しています。ファイルは外部に送信されません。」）
- 右下隅トースト「Escを2連打で擬態モードへ」
- 右上に「設定」「小窓表示」「言語切替（日/EN）」ボタン
- **フッター（追加必須）**：`キノメノ：https://note.com/kinomeno/xxxxx ☞ko-fiで支援 ☞noteでチップ`

ファイル投入後の画面もchatGPTから提案あり、採用。

### 3.8.1 標準モード（投入後）レイアウト

3カラム構成。

- **左カラム**：
  - 上部：小さなドロップ領域「追加のファイルをここにドラッグ＆ドロップ または クリックして選択」（仕様：継続投入用）
  - 中部：処理状況パネル（処理完了表示、処理したファイル数／スキップ数／生成PDF数）
  - 下部：「対応していないファイルは○件スキップしました 詳細を見る」（折りたたみ）
- **中央カラム**：
  - ヘッダー「生成結果（N件）」＋緑の「全て印刷」ボタン
  - ファイルカード一覧（投入順）：アイコン（PDF=赤／画像=緑／テキスト=黄緑、拡張子で自動判別）、ファイル名、ページ数、種類タグ、サムネ画像（一部ページ）、「印刷」「PDFを保存」ボタン、ケバブメニュー
  - 注記「※ サムネイルは一部ページのイメージです」（長尺PDF全ページ描画を避ける逃げ口）
- **右カラム**：
  - 設定パネル：ダミー濃度スライダー（薄い/普通/濃い/狂気）、画面モード切替（無料2＋有料3）、有料パターン解放（パスワード入力＋解除＋購入リンク）、その他オプション（両面推奨メッセージ表示、生成後プレビュー）
  - 印刷ログ（直近）パネル：時刻＋ファイル名＋状態（印刷完了／PDFを保存）。「すべてのログを見る」リンク
- 下部固定：プライバシー明示バー＋「Esc 2連打で擬態モードへ」トースト＋フッター（キノメノ＋ko-fi/note）

### 3.8.2 擬態モード「スプレッドシート風」レイアウト

- 上部：スプレッドシート風メニューバー（ファイル／編集／表示／挿入／データ／ツール／ヘルプ／自動保存済み）
- ツールバー：書式・フォントサイズ・B/I/S・罫線・色などのアイコン群（見た目だけ）
- 本体：ヘッダ行（A=ファイル名、B=ページ数、C=種類、D=プレビュー、E=印刷、F=PDFを保存、J=ステータス、I=更新日時）＋データ行
- 印刷ボタンは緑のプリンタアイコンとしてセルに溶け込む
- 右に設定パネル「擬態中」表示で状態を見失わせない
- 下部固定：プライバシーバー＋フッターは継続表示
- 採用判断：実装後、設定パネルの常時表示は擬態度を弱めるので「セルに偽装したリンクで開閉」する案も検討する

### 3.8.3 小窓モード

参考イメージ：`GPTIMAGE/IMG_5876_note.png`（3画面構成内、左下に描かれた小窓パネル）。

- 画面の隅（左下 or 右下）に小さな半透明パネルとして表示。
- 内容：縮小ロゴ＋ファイル一覧の最小表示（ファイル名＋小アイコン）＋必要最小限のアクションボタン。
- 背景は透過 or 半透明で、他ウィンドウに重なっても邪魔にならない。
- 再度ボタンで元の標準モードに戻す。
- 実装はCSS（`position: fixed; bottom; right;` ＋サイズ縮小＋要素間引き）で対応。ブラウザ縮小APIは使わない。

## 4. スコープ

### Lv1（今回作る）
- ファイル投入（PDF / 画像 / txt / 貼り付け、複数・フォルダ対応、サブフォルダは非再帰）

- 生成結果の一覧表示、個別印刷／全部印刷
- 対応外スキップ・大量時警告
- カモフラージュ画面モード（無料2種＋有料解放枠、Esc2連打、ウィンドウ縮小）
- 無料／有料コード分離
- ロゴ（エコ/eco を大きく）・キャッチ（私の極小）

### Lv2（可能なら）
- 印刷ダイアログを開くところまで自動化

### スコープ外
- Lv2.5 ブラウザ拡張版、Lv3 exe版
- Excel / Word / eml 対応
- ただし**裏ダミー生成エンジンは独立モジュール化**し、将来共有可能にする
